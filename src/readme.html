How to Use:
Using SubEdit is as easy as dragging and dropping your .mkv or raw subtitle file onto SubEdit.bat and letting it run.
If you do not have Python installed, you can get Python here. Adding python.exe to your path environment variable is the easiest way to ensure SubEdit.py works.
The documentation below is if you want to modify or create your own script.

Examples:
Most of the syntax can be easily learned from looking at a few examples.
Each modifier rule is written on top, with a short description of how it works below it.
word -> another word
Replace all instances of "word" with "another word".
Examples:
"a word in a sentence" -> "a another word in a sentence"
"a compoundword here" -> "a compoundword here"
* "word" -> "another word"
Replace words ending with "word" with "another word".
Examples:
"a word in a sentence" -> "a another word in a sentence"
"a compoundword here" -> "a another word here"
* "word" -> * "another word"
Replace words ending with "word" with "another word", keeping prefix.
Examples:
"a word in a sentence" -> "a another word in a sentence"
"a compoundword here" -> "a compoundanother word here"
* "word" * -> * "phrase" * !
Replace words ending containing "word" with "phrase", keeping prefix, suffix, and matching case.
Examples:
"a word in a sentence" -> "a phrase in a sentence"
"a compoundword here" -> "a compoundphrase here"
"a Word in a sentence" -> "a Phrase in a sentence"
"a compoundWORDcompound here" -> "a compoundPHRASEcompound here"
'word' -> * 'phrase' * ?
Replace instances of "word" with "phrase", matching case.
The *'s in the second half mean nothing in this case.
Examples:
"a word in a sentence" -> "a phrase in a sentence"
"a Word in a sentence" -> "a Word in a sentence"
"a compoundword here" -> "a compoundword here"
/(word)|(phrase)/i -> "{1CX:no}{2CX:yes}"
Replace "word" with "no", "phrase" with "yes", matching case.
Examples:
"a word in a sentence" -> "a no in a sentence"
"a WORD in a sentence" -> "a NO in a sentence"
"a phrase" -> "a yes"
"a Phrase" -> "a Yes"
/(\w+)/i -> "\{a{1C:b\{\"c\"\}d}e\}"
Replace all words with '{ab{"c"}de}'.
This shows how escapes work by using the backslash.
Example:
'word Word WORD' -> '{ab{"c"}de} {Ab{"c"}de} {AB{"C"}DE}'
Command Line:
The general form of the command line is:
SubEdit.py [options] [input_files]
input_files is a list of files which are to be used as input.
Valid file extensions for subtitles are: .mkv, .ssa, .ass, .srt
Valid file extension for settings is: .txt
Any other extension will throw an error.
Options can be divided up into 2 categories: general options and declaration options.
General options begin with a single hypen ("-"), followed by the option name. The following general options are available:
-s file
-
Set the file to be used as the settings file.
Aliases: -settings file
-se file
-
Set the file to be used as the settings file if the file exists. Otherwise, this option is ignored.
Aliases: -settings_if_exists file
-i file
-
Specify an input file to modify.
Unlike the [input_files] part of the command line, this option accepts any file extension without question.
Aliases: -input file
-ie file
-
Specify an input file to modify so long as the file exists.
Unlike the [input_files] part of the command line, this option accepts any file extension without question.
Aliases: -input_if_exists file
-v
-
Display the version info of SubEdit and terminate.
Version info is formatted like: Version X.Y.Z
Aliases: -version
Declarative options begin with two hypens ("--"), followed by the script variable name. See Script Variables for more information.
Script Variables:
Script control variables can be set using declarative comments as described below.
The available variables are:
group
-
Modifiy the release group name according to this string.
* are replaced with the original group name.
Default: "*-Sedit"
script
-
A modified name of the subtitles track.
* are replaced with the original subtitle track name.
Default: "* [SubEdited]"
crc
-
Update the CRC field of the filename for the resulting file.
"true" - Updates the CRC
"false" - Does not update the CRC
Default: "true"
internal
-
When modifying .mkv files, this option tells whether or not the new subtitles should be re-mux'd into a new .mkv file.
"true" - Create a new .mkv file with new subtitles
"false" - Leave new subtitles as an external file
Default: "true"
keep_original
-
When modifying .mkv files, this option tells whether or not the original subtitles should be kept (but not as the main subtitle track.)
"true" - Keep original subtitles
"false" - Remove original subtitles
Default: "true"
debug
-
Outputs the results of subprocess calls when enabled.
"true" - Debug enabled
"false" - Debug disabled
Default: "false"
Script Syntax:
The basic format of settings files is as follows:
Each line is composed of an optional modifier declaration, optionally followed by a comment.
line ::= [modifier] [comment]
Script Comments:
Comments have two styles: descriptive and declarative. Declarative comments are formatted as follows:
#@variable=value
comment_decl ::=
"#@" comment_decl_var "=" comment_decl_value
comment_decl_var ::=
string
comment_decl_value ::=
string
Whereas descriptive comments are everything else beginning with a #:
# some comment about the script
comment_desc ::=
"#@" string
Modifier Formats:
Modifier patterns have 2 general formats: normal, and regular expressions.
The type is determined by the first non-whitespace character. If it is a "/", it is a regex modifier, otherwise it's a normal modifier.
Normal modifier patterns can look like:
word1 -> replacement1
*"word2" -> *"replacement2"
*'word3'* -> *'replacement3'* !
*'word4'* -> "replacement4" !?
'word4'* -> "replacement4"* ?
"word5"* -> "replace with \"quotes\"" ?
'word5' -> 'replace with \\ single backslash' ?!
modifier ::=
mod_search "->" mod_replace
mod_search ::=
mod_part
mod_replace ::=
mod_part mod_flags
mod_part ::=
["*"] (mod_part_q | mod_part_unq) ["*"]
mod_part_q ::=
"\"" escaped_string "\"" | "'" escaped_string "'"
mod_part_unq ::=
string
mod_flags ::=
("?" | "!")*
Regular expression patterns can look like:
/word1/ -> replacement1
/word2/i -> "replacement2"
/re(g)ex/i -> "replacement3 {1}"
/re(g)ex/i -> "replacement4 {1:words}"
/re(g)ex/i -> "replacement5 {1C:modified}"
/re(g)ex/i -> "\{escapes\} {1C:more\{escapes\}}"
re_modifier ::=
re_mod_search "->" re_mod_replace
re_mod_search ::=
"/" regex "/" regex_flags
re_mod_replace ::=
"\"" regex_fmt "\"" | "'" regex_fmt "'" | regex_unfmt
regex ::=
// Basic regex format; similar to Javascript regex literals
regex_flags ::=
regex_flag*
regex_flag ::=
("I" | "L" | "M" | "S" | "X")
regex_fmt ::=
escaped_string
regex_unfmt ::=
string
Regular Expressions:
The regex syntax is the same as Python's regex syntax.
Regex patterns are formatted very similar to how Javascript regex literals look, and Python regex and Javascript regex are pretty similar anyways. The only difference is the flags at the end of the expression.
Regular Expression Flags:
The following are regex_flags. I is the most useful one, the others are included just in case they are needed.
The flags are also case insensitive.
I
-
Ignore case
-
Text case will be ignored when matching
L
-
Locale
-
Makes certain matchers depend on locale
M
-
Multi-line
-
^ matches line start, $ matches line end
S
-
Dot all
-
. can match newlines
X
-
Verbose
-
Allows expressions to be written differently
Regex Replacement Format:
The regex format is used to replace regex matches with a new text, including being able to use matched groups in the replacement.
The general format is:
"some text {group_mod} some text ..."
Anything following a backslash (\) is escaped, both inside and outside of groups.
{1}
{1:text}
{1C:text}
{1X:text}
group_mod ::=
"{" group_id [group_flags] [":" group_text] "}"
group_id ::=
number
group_flags ::=
("C" | "I" | "L" | "P" | "U" | "X" | "x")
group_text ::=
string
If an unquoted regex_unfmt is used, direct replacement will be used without parsing for groups.
Regex Replacement, [:group_text] omitted:
When group_text is omitted, the match group with group_id is used as the replacement.
group_id=0 is the entire match, 1 is the first group, 2 is the second, etc.
group_flags meanings:
C
-
Capitalized version of the match group
I
-
Invert the case of the match group
L
-
Lower-case version of the match group
P
-
Proper-Case version of the match group
U
-
Upper-case version of the match group
X
-
Meaningless
x
-
Meaningless
Regex Replacement, [:group_text] included:
When group_text is not omitted, the group_text is used as the replacement text instead of match group. It is modified according to group_flags.
group_flags meanings:
C
-
group_text capitalization mimics match group
I
-
Invert the case of the group_text
L
-
Lower-case version of group_text
P
-
Proper-Case version of group_text
U
-
Upper-case version of group_text
X
-
Omit replacement if match group is null
x
-
Omit replacement if match group is not null
